VI. Java Operators  
 
In Java programming, operators are special symbols that perform  operations on one or more operands 
(values  or  variables).  They  are  fundamental  building  blocks  for  constructing  expressions  and  controlling 
program flow. Understanding operators is crucial for   writing effective and efficient Java code, enabling you to 
manipulate data, make decisions, and perform calculations.  
 
This lesson will provide a comprehensive overview of various operator types in Java, their functionalities, and 
the rules governing their evaluation. By the end of this chapter, you will be equipped with the foundational 
programming skills necessary to eff ectively utilize operators in your Java applications.

2.   Increment Operator (++)  
 
The increment operator increases the value of its operand by 1. It can be used in two forms:  
•   Prefix increment (++variable): Increments the value before using it in the expression.  
•   Postfix increment (variable++): Increments the value after using it in the expression.  
 
Example:  
 
public class IncrementOperator {  
     public static void main(String[] args) {  
         int x = 5;  
         int y = 5;

3 . Decrement Operator ( -- )  
 
The decrement operator decreases the value of its operand by 1. Similar to the increment operator, it also 
has prefix and postfix forms:  
•   Prefix decrement ( -- variable): Decrements the value before using it in the expression.  
•   Postfix decrement (variable -- ): Decrements the value after using it in the expression.  
 
Example:  
 
public class DecrementOperator {  
     public static void main(String[] args) {  
         int x = 10;  
         int y = 10;  
 
         // Prefix decrement  
         int result1 =  -- x; // x becomes 9, then result1 becomes 9  
         System.out.println("Prefix Decrement: x = " + x + ", result1 = " + result1); // Output: x = 9, result1 = 9  
 
         // Postfix decrement  
         int result2 = y -- ; // result2 becomes 10, then y becomes 9  
         System.out.println("Postfix Decrement: y = " + y + ", result2 = " + result2); // Output: y = 9, result2 = 10  
     }  
}

Short - Circuiting:  
 
•   && (Logical AND) is  a short - circuit operator. If the left operand is false, the right operand is not

7. Rules of Precedence  
 
When multiple operators are present in an expression, Java follows a strict set of rules, known as operator 
precedence, to determine the order in which operations are performed. Operators with higher precedence 
are  evaluated  before  operators  with  lower  pre cedence.  If  operators  have  the  same  precedence,  their 
associativity (left - to - right or right - to - left) determines the order.  
 
Parentheses ()   can be used to override the default precedence, forcing certain operations to be evaluated 
first.  
 
Common Operator Precedence (Highest to Lowest):  
•   Postfix operators: expr++, expr --  
•   Prefix operators: ++expr,  -- expr, +expr,  - expr, ~, !  
•   Multiplicative: *, /, %  
•   Additive: +,  -  
•   Shift: <<, >>, >>>  
•   Relational: <, >, <=, >=, instanceof  
•   Equality: ==, !=  
•   Bitwise AND: &  
•   Bitwise XOR: ^  
•   Bitwise OR:  \ |  
•   Logical AND: &&  
•   Logical OR: ||  
•   Ternary (Conditional): ? :  
•   Assignment: =, +=,  - =, *=, /=, %=, &=, ^=, |=, <<=, >>=, >>>=

8.  Type Casting in Java  
 
Type casting   is the process of converting a value from one data type to another. Java supports two types of 
casting: implicit and explicit. This concept is crucial for managing data types effectively and avoiding potential 
data loss.  
 
Implicit Type Casting (Widening Conversion)  
 
Implicit type casting, also known as widening conversion, occurs automatically when:  
•   Converting a smaller data type to a larger data type.  
•   There is no loss of data.  
•   The target type can accommodate the source type's value without any precision issues.  
 
Order of widening conversion:  
 
128

byte  - > short  - > char  - > int  - > long  - > float  - > double  
 
Example:  
 
public class ImplicitCasting {  
     public static void main(String[] args) {  
         int myInt = 100;  
         double myDouble = myInt; // Implicit casting from int to double  
         System.out.println("Original int: " + myInt);    // Output: 100  
         System.out.println("Implicitly casted double: " + myDouble); // Output: 100.0  
 
         char myChar = 'A';  
         int charToInt = myChar; // Implicit casting from char to int (ASCII value)  
         System.out.println("Original char: " + myChar);  // Output: A  
         System.out.println("Implicitly casted int from char: " + charToInt); // Output: 65  
     }  
}  
 
Explicit Type Casting (Narrowing Conversion)  
 
Explicit type casting, also known as narrowing conversion, occurs when:  
•   Converting a larger data type to a smaller data type.  
•   It requires a cast operator () to explicitly tell the compiler that you intend to perform the conversion.  
•   There is a potential for data loss or precision issues if the value of the larger type exceeds the range 
of the smaller type.  
 
Example:  
 
public class ExplicitCasting {  
     public static void main(String[] args) {  
         double myDouble = 10.99;  
         int myInt = (int) myDouble; // Explicit casting from double to int (truncates decimal)  
         System.out.println("Original double: " + myDouble);  // Output: 10.99  
         System.out.println("Explicitly casted int: " + myInt); // Output: 10 (data loss: .99 is lost)  
 
         int largeInt = 200;  
         byte myByte = (byte) largeInt; // Explicit casting from int to byte  
         System.out.println("Original int: " + largeInt); // Output: 200  
         System.out.println("Explicitly casted byte: " + myByte); // Output:  - 56 (data loss: byte range is  - 128 to 127, 
overflow occurs)  
 
         // Example of precision loss with float/double  
         float f = 123.456789f;  
         double d = f; // Widening, no immediate loss  
         System.out.println("Float: " + f); // Output: 123.45679

Potential Data Loss or Precision Issues:  
•   Truncation .   When casting a floating - point number (float or double) to an integer type (byte, short, 
int, long), the decimal part is simply truncated, not rounded.  
•   Overflow/Underflow .   When casting a value from a larger integer type to a smaller integer type, if the 
value exceeds the range of the smaller type, it will wrap around (overflow or underflow), leading to an 
incorrect value.  
•   Loss of Precision .   While casting from float to double is widening, casting from double to float can 
result in loss of precision, as float has fewer significant digits than double. Similarly, very large long 
values might lose precision when cast to float or double due to the ir internal representation.
