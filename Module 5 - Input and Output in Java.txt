V.  Input and Output in Java  
 
In the world of programming, interaction is key. Programs are rarely isolated entities; they often need to 
communicate with the outside world. This communication happens through Input/Output (I/O) operations. 
Input   allows a program to receive data from sources like the keyboard, files, or networks, making it dynamic 
and responsive.  Output   enables a program to display information to users, save data to files, or send data 
over a network, making its results visible and useful.  
 
This chapter will guide you through the fundamental concepts of I/O in Java, focusing on how to display 
information to the console and how to read various types of data from the user's keyboard. Mastering these 
operations is crucial for building interactiv e and practical Java applications.

1.   Output Operations  
The most common way for a Java program to display information to the console is by using the System.out 
object. This object provides several methods for printing.  
 
System.out.println()  
 
The  println()   method (short for "print line") prints the given argument to the console and then moves the 
cursor to the next line.  
 
Syntax:  
 
System.out.println(data);  
 
Where data can be a string, number, boolean, or any other data type.  
 
Example:  
 
public class OutputExample {  
     public static void main(String[] args) {  
         System.out.println("Hello, World!"); // Prints "Hello, World!" and moves to a new line  
         System.out.println(123);             // Prints 123 and moves to a new line  
         System.out.println(true);            // Prints true and moves to a new line  
     }  
}  
 
Output:  
 
Hello, World!  
123  
true  
 
System.out.print()  
 
The  print()   method is similar to println(), but it does not add a new line character at the end. The cursor 
remains on the same line after printing.  
 
Syntax:  
 
System.out.print(data);  
 
Example:

3. String Concatenation  
 
String concatenation   is the process of joining two or more strings together to form a single string. In Java, 
you can use the  +  operator to concatenate strings. When you use the  +   operator with a string and another 
data type (like an integer or a double), Java automatically converts the non - string data type into its string 
representation and then concatenates it.  
 
public class ConcatenationExample {  
     public static void main(String[] args) {  
         String firstName = "Maria";  
         String lastName = "Dela Cruz";  
         int age = 25;  
         double height = 1.65;  
 
         // Concatenating strings and variables  
         System.out.println("Full Name: " + firstName + " " + lastName);  
         System.out.println("Age: " + age + " years old.");  
         System.out.println("Height: " + height + " meters.");  
 
         String message = "Welcome, " + firstName + "! Your age is " + age + ".";  
         System.out.println(message);  
     }  
}  
 
Output:  
 
Full Name: Maria Dela Cruz  
Age: 25 years old.  
Height: 1.65 meters.  
Welcome, Maria! Your age is 25.

In Java, the  Scanner  class is the most common and convenient way to achieve this.

The   Scanner  class is part of the java.util package, so you need to import it before use.

import java.util.Scanner;

Scanner input = new Scanner(System.in);

Comparing next() vs. nextLine()  
 
This is a common point of confusion for beginners.  
 
next()   -     Reads input up to the first whitespace character (space, tab, newline). It only returns a single 
"word" or "token". After reading, the cursor remains at the whitespace character.  
 
nextLine()   -   Reads  the  entire  line  of  input,  including  any  spaces,  until  it  encounters  a  newline 
character ( \ n). It then consumes the newline character and moves the cursor to the next line.  
 
The "Newline Trap":  
When you use nextInt(), nextDouble(), nextBoolean(), etc., they read only the number/boolean and 
leave the newline character ( \ n) in the input buffer. If you then immediately call nextLine(), it will read 
this leftover newline character as an empty string  and skip waiting for user input.  
 
Solution: After calling nextInt(), nextDouble(), etc., and before calling nextLine(), you should add an 
extra scanner.nextLine(); to consume the leftover newline character.  
 
Example of the Newline Trap and its Solution:  
 
import java.util.Scanner;  
 
public class NewlineTrapExample {  
     public static void main(String[] args) {  
         Scanner scanner = new Scanner(System.in);  
 
         System.out.print("Enter your age: ");  
         int age = scanner.nextInt(); // Reads 30, leaves ' \ n' in buffer  
         // If you enter 30 and press Enter, the input buffer looks like "30 \ n"  
         // nextInt() reads "30", leaving " \ n"  
 
         // PROBLEM: If you uncomment the next line, it will read the leftover ' \ n'  
         // System.out.print("Enter your full name (PROBLEM): ");  
         // String fullNameProblem = scanner.nextLine(); // Reads the leftover ' \ n' immediately!  
         // System.out.println("Problematic Full Name: " + fullNameProblem);  
 
 
         // SOLUTION: Consume the leftover newline character  
         scanner.nextLine(); // Consumes the ' \ n' left by nextInt()  
 
         System.out.print("Enter your full name (SOLUTION): ");  
         String fullNameSolution = scanner.nextLine(); // Now it waits for actual input
