LETTER TO THE READERS   AND USERS  
 
Welcome to COMP 102: Computer Programming 1 — a foundational course 
designed to equip you with the essential skills in programming using Java, one of 
the world’s most widely used and versatile programming languages.   
 
This module acts as your entry point into programming, providing you with 
essential  theoretical  knowledge  and  practical  skills  necessary  to  begin  your 
journey  as  a  developer.  Whether  your  goal  is  software  development,  systems 
analysis, data science, or any   computing - related field, the concepts and exercises 
included here will lay the foundation for your technical expertise.  
 
Throughout this module, you will learn how to write, compile, and run Java 
programs, understand fundamental programming constructs such as variables, 
data  types,  control  structures,  and  arrays.  Our  goal  is  to  make  programming 
approachable, engaging, and pr actical so you can apply these skills to real - world 
problems.   Specifically, this module will cover the following topics:  
 
Introduction to Computer Programming   –   Understand the basic  components  
of computers, and various programming paradigms and languages.  
 
Problem  Solving  and  Software  Development   –   Learn  how  to  break  down 
complex problems and apply the Software Development Life Cycle (SDLC).  
 
Introduction  to  Flowcharting  and  Programming   –   Develop  logical  solutions 
using pseudocode and flowcharts to build a solid foundation for coding.  
 
Introduction to Java   La ng uage and Programming Fu ndamentals   –   Dive into 
the  language’s  history,  features,  development  environment,  and  how  Java 
programs are built and executed.   Gain proficiency in syntax, data types, variables, 
constants, operators, and memory handling.  
 
Input and Outpu t in Java –   Master basic console input and output operations 
including formatting, escape sequences, and placeholders.

Java  Operators   –   Perform arithmetic and logical operations and convert data 
types appropriately.  
 
Strings  and  Built - in  Libraries   –   Work  with  text  and  use  built - in  functions  to 
manipulate data more effectively.  
 
Selection  and Iterative Structures   –   Implement decision - making using selection 
structures and loops to control the flow of your programs.  
 
Arrays  –   Store and manage collections of data efficiently using one - dimensional 
and multi - dimensional arrays.  
 
Our goal is to help you develop computational thinking, strengthen your 
logical reasoning, and enable you to construct meaningful solutions using Java.  
 
As you progress through the lessons,  I   encourage you to actively participate 
in hands - on activities, engage with practice problems, and work on programming 
exercises provided in both lecture and laboratory settings.  
 
This module is aligned with outcome - based education principles, ensuring 
that every activity and assessment is designed to foster higher - order thinking, 
problem - solving ability, and practical application of knowledge.  
 
I   hope this module serves as a strong foundation for your future studies 
and inspires you to further explore the exciting field of software development.

programming is to create a set of instructions that tell a computer how t o perform a specific task or solve a

1.   The Computer System  
 
A  computer  is an electronic data processing device, which accepts and stores data input, processes the data 
input, and generates the output in a required format.  
 
A  computer system   is an integrated system consisting of hardware and software designed to process data 
into information.  
 
Components of a Computer System  
 
•   Hardware  –   Physical components (CPU, RAM, storage, input/output devices).  
•   Software  –   Programs that instruct hardware (Operating System, applications).  
•   Data  –   Raw facts processed by the computer.  
•   User  –   The person interacting with the computer.

2.   Analog vs. Digital Computers  
 
Analog  computers  are  devices  that  perform  calculations  and  process  data  using  continuous  physical 
quantities. These quantities can vary smoothly within a range and are used to model or simulate real - world 
phenomena.  
 
How They Work  
 
•   They use physical quantities such as electrical voltages, currents, mechanical movements, or fluid 
pressures.  
•   Data is represented by the magnitude of these quantities, which can take on any value within a given 
range.  
•   Operations are performed through the physical manipulation of these quantities using components 
like resistors, capacitors, amplifiers, and mechanical parts.  
 
Applications  
 
•   Historically  used  in  scientific  and  engineering  calculations  such  as  differential  equations,  flight 
simulations, and control systems.  
•   Examples include slide rule, voltmeters, oscilloscopes, and early analog computers like the Differential 
Analyze  
7

Advantages  
 
•   Capable  of  real - time  processing  of  continuous  data,  making  them  suitable  for  dynamic  system 
simulations.  
•   Faster for certain types of calculations involving physical systems.  
•   Useful in applications where approximate solutions are sufficient.  
 
Disadvantages  
 
•   Limited accuracy due to noise, signal degradation, and component tolerances.  
•   Difficult to store, reproduce, or modify data once set.  
•   Not easily programmable or adaptable to different tasks.  
•   Large size and maintenance requirements.  
 
Digital  computers   process  data  in  discrete  steps  using  binary  code  (combinations  of  0s  and  1s).  They 
manipulate digital signals that represent information in binary form.  
 
How They Work  
•   Data is encoded as binary numbers.  
•   They perform operations using logic gates, microprocessors, and memory units.  
•   Instructions are stored as programs, allowing for complex, flexible, and precise operations.  
 
Applications:  
•   Used  in  virtually  all  modern  technology:  personal  computers,  smartphones,  servers,  embedded 
systems, and more.  
•   Suitable for tasks requiring high accuracy, data storage, complex calculations, and automation.  
 
Advantages  
•   High precision and accuracy in calculations.  
•   Easy to store, retrieve, and manipulate data electronically.  
•   Highly programmable and adaptable to a wide range of tasks.  
•   Less susceptible to noise and external interference.  
•   Compact and energy - efficient compared to analog counterparts.  
 
Disadvantages:  
•   Can be slower in real - time processing of continuous signals compared to analog systems.  
•   Complexity and cost can be higher for certain specialized applications.  
•   Requires software and programming expertise.  
 
3.   Programming Languages  
 
A  programming language   is a formal set of instructions used to communicate with a computer and create 
software programs. They allow humans to write code that computers can execute.  
 
8

COMPUTER PROGRAMMING 1  
 

 
Example: Visual Basic, C++, C#, Java, Assembly Language  
 
Three   fundamental concepts   of programming language:   syntax, grammar, and semantics.  
•   Syntax -   refers to the set of rules that define the correct structure of statements or expressions in a 
programming language. It governs how symbols, keywords, and other elements should be arranged 
to form valid and meaningful code.  
•   Semantics  -   Semantics deals with the meaning or interpretation of code. It focuses on whether the 
code, once executed, behaves as intended and produces the desired results.   
•   Grammar   -   refers  to  the  rules  that  define  the structure  and  composition of  valid  statements  or 
expressions in a programming language. It encompasses both syntax and semantics, providing a 
comprehensive set of rules for how code should be written and how it should   behave.  
 
Uses  o f Programming Languages  
 
Programming languages are used for a wide range of applications across many industries, including:  
•   Software  Development.   Programming  languages  are  primarily  used  for  software  development. 
Developers use programming languages to write code for applications, systems software, and other 
computer programs.  
•   Web  Development.   Programming  languages  like  HTML,  CSS,  and  JavaScript  are  used  for  web 
development to create dynamic and interactive websites.  
•   Data Science and Analytics.   Programming languages like Python and R are popular in data science 
and analytics to analyze and interpret data, and to build machine learning models.  
•   Scientific  Computing.   Programming  languages  like  Fortran  and  MATLAB  are  used  for  scientific 
computing to solve complex mathematical and scientific problems.  
•   Game Development.   Programming languages like C++ and C# are used for game development to 
create video games and other interactive media.  
•   Mobile  App  Development.   Programming  languages  like  Java and  Swift  are  used  for  mobile  app 
development to create apps for iOS and Android platforms.  
•   Artificial  Intelligence  and  Robotics.   Programming  languages  like  Python  and  C++  are  used  in 
artificial intelligence and robotics to build intelligent systems that can learn, reason, and interact with 
the physical world.  
•   Embedded Systems.   Programming languages like C and Assembly are used for embedded systems 
development to create software for small, low - power devices like microcontrollers and IoT devices.  
 
Categories of Programming Languages  
 
Machine Language  
 
The lowest level programming language, consisting entirely of binary 
code (0s and 1s) that the computer’s CPU directly reads and executes.  
 
Characteristics:  
Figure  11.  Machine  Code  •   Programs  are  sequences  of  machine  instructions  in  binary 
Representation of z = x + y   form.

•   Instructions specify operations (like addition, data movement) and memory locations involved.  
•   Machine language is hardware - specific; code written for one CPU architecture won’t run on another.  
 
Advantages:  
•   Directly executed by the CPU, making it extremely fast and efficient.  
 
Disadvantages:  
•   Difficult for humans to read, write, and debug.  
•   Highly error - prone due to its complexity and lack of abstraction.  
 
Assembly Language  
 
A symbolic representation of machine language specific to a processor architecture.  
 
Characteristics:  
•   Uses mnemonics (short, human - readable codes) for instructions (e.g., MOV, ADD).  
•   Each line of assembly code usually corresponds to one machine instruction (one - to - one).  
•   Requires an assembler to convert assembly code into machine code.  
•   Allows programmers to work with hardware efficiently but is still low - level and hardware dependent.  
 
Advantages:  
•   More readable than machine code.  
•   Provides precise control over hardware and system resources.  
 
Disadvantages:  
•   Programming is slow and error prone.  
•   Not portable across different CPU architectures.

High Level  Programming Language  
 
Languages designed to be easy for humans to read, write, and understand, abstracting away hardware 
details.  
 
Characteristics:  
•   Use natural language - like syntax and constructs (e.g., loops, functions).  
•   Programs must be translated into machine code via a compiler or interpreter before execution.

Advantages:  
•   Highly portable across different hardware platforms.  
•   Easier and faster to develop, maintain, and debug.  
 
Disadvantages:  
•   Typically, slower than low - level languages due to abstraction overhead.  
•   Less direct control over hardware.

Source Program  
 
The source program is the original code written by the programmer in a programming language.   It is 
human - readable and saved in files with specific extensions, for example:  
 
•   Pascal   -   . pas  
•   C++   -     .c, .cpp, .h  
•   Java   -   .java  
•   Python  -   .py  
 
Source programs are either compiled into machine code or executed by an interpreter.  
 
 
Compiler  
 
A   compiler   is a software tool that translates a source program written in a high - level language into machine 
language (object code).   Some compilers generate assembly language first, which is then converted to

machine code by an assembler.   Compilers typically translate the entire program before execution, enabling 
faster runtime performance.   Each high - level statement often corresponds to multiple machine instructions.  
 
Interpreter  
 
An  interpreter   translates and executes source code line - by - line at runtime.   Commonly used for scripting 
languages like Python, Ruby, and JavaScript.    Interpreted programs are generally slower than compiled ones 
but offer greater flexibility and easier debugging.   Interpreters do not produce a separate machine code file; 
they execute code directly.  
 
4.   Scripting Languages  
 
Scripting languages   are programming languages designed for integrating and communicating with other 
programming languages. They are often used to automate tasks, manipulate data, and control software 
applications.  
 
Key features:  
 
•   Interpreted .  Scripts are executed directly by an interpreter rather than compiled into machine code.  
•   Ease of Use .  Designed to be simple, flexible, and quick to write.  
•   Automation .  Ideal for automating repetitive tasks.

6. Number System  
 
When  we  type  some  letters  or  words,  the  computer  translates  them  in  numbers  as  computers  can 
understand only numbers. A computer can understand the positional number system where there are only 
a few symbols called  digits,   and these symbols represent different values depending on the position they 
occupy in the number.  
 
The value of each digit in a number can be determined using −  
•   The digit  
•   The position of the digit in the number  
•   The base of the number system (where the base is defined as the total number of digits available in 
the number system)  
 
Decimal Number System  
 
The number system that we use in our day - to - day life is the decimal number system.  Decimal number 
system   has base 10 as it uses 10 digits from 0 to 9. In decimal number system, the successive positions to 
the left of the decimal point represent units, tens, hundreds, thousands, and so on.  
 
Each position represents a specific power of the base (10). For example, the decimal number 1234 consists 
of the digit 4 in the units position, 3 in the tens position, 2 in the hundreds position, and 1 in the thousands 
position. Its value can be written as  
 
(1 x 1000)+ (2 x 100)+ (3 x 10)+ (4 x l)  
(1 x 103)+ (2 x 102)+ (3 x 101)+ (4 x l00)  
1000 + 200 + 30 + 4  
1234

7.  Number  System  Conversion  
 
There are many methods or techniques which can be used to convert numbers from one base to another.  
 
Decimal to Other Base System  
Other Base System to Decimal  
Other Base System to Non - Decimal  
Shortcut method  -   Binary to Octal  
Shortcut method  -   Octal to Binary  
Shortcut method  -   Binary to Hexadecimal  
Shortcut method  -   Hexadecimal to Binary  
 
Decimal to Other Base System  
 
Step 1 − Divide the decimal number to be converted by the value of the new base.  
Step 2 − Get the remainder from Step 1 as the rightmost digit (least significant digit) of the new base number.  
Step 3 − Divide the quotient of the previous divide by the new base.  
Step 4 − Record the remainder from Step 3 as the next digit (to the left) of the new base number.

Repeat Steps 3 and 4, getting remainders from right to left, until the quotient becomes zero in Step 3.  
The last remainder thus obtained will be the Most Significant Digit (MSD) of the new base number.  
 
 
 
Figure 20.    Conversion of decimal 29   to binary.   10
 
Other Base System to Decimal System  
 
Step 1 − Determine the column (positional) value of each digit (this depends on the position of the digit and 
the base of the number system).  
Step 2 − Multiply the obtained column values (in Step 1) by the digits in the corresponding columns.  
Step 3 − Sum the products calculated in Step 2. The total is the equivalent value in decimal.
