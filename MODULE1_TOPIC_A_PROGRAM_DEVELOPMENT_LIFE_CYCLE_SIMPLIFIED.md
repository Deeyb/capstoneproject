# Program Development Life Cycle

The **Program Development Life Cycle** (PDLC) is a step-by-step process used to develop computer programs. It helps organize the work needed to create software.

## What is PDLC?

- A model that shows the stages and tasks needed to create software
- A set of phases used to develop a program in any programming language

---

## The 6 Phases

1. **Problem Definition** - What problem do we need to solve?
2. **Problem Analysis** - What do we need to solve it?
3. **Algorithm Development** - How do we solve it step-by-step?
4. **Coding & Documentation** - Write the program code
5. **Testing & Debugging** - Check if it works correctly
6. **Maintenance** - Update and improve the program

---

## Phase 1: Problem Definition

**Goal:** Understand what problem we need to solve

**What to do:**
- Clearly state the problem in simple English
- Understand what the solution should do
- Focus on the main problem, ignore unimportant details
- Ask questions if something is not clear

---

## Phase 2: Problem Analysis

**Goal:** Figure out what we need to solve the problem

**What to do:**
- List all the inputs needed (what the user will enter)
- List all the outputs needed (what the program will show)
- Identify any formulas or calculations needed
- Decide what type of data we're working with (numbers, text, etc.)

---

## Phase 3: Algorithm Development

**Goal:** Create a step-by-step plan to solve the problem

**Also called:** Design Phase

**What to do:**
- Write the steps to solve the problem (this is called an "algorithm")
- Use pseudocode, flowcharts, or IPO charts
- Test your plan with sample inputs to make sure it works

**Remember:** Use Top-Down design - start with big steps, then break them into smaller steps

---

## Phase 4: Coding and Documentation

**Goal:** Write the actual program code

**What to do:**
- Use a programming language (like C++, Java, Python) to write the program
- Follow the algorithm you created in Phase 3
- Write comments in your code to explain what it does (documentation)

---

## Phase 5: Testing & Debugging

**Goal:** Make sure the program works correctly

**What to do:**
- Test the program with different input values
- Check if it gives the correct output
- Fix any errors (bugs) you find
- Test again until everything works

---

## Phase 6: Maintenance

**Goal:** Keep the program working and improve it

**What to do:**
- Use the program in real situations
- If users find problems, fix them
- If users want new features, add them
- When making changes, go back through the phases as needed

---

## Summary

The Program Development Life Cycle has **6 phases** that guide you from understanding a problem to creating a working program:

1. **Problem Definition** → Know what to solve
2. **Problem Analysis** → Know what you need
3. **Algorithm Development** → Plan how to solve it
4. **Coding & Documentation** → Write the code
5. **Testing & Debugging** → Make sure it works
6. **Maintenance** → Keep it working and improve it

Each phase builds on the previous one, so it's important to complete each step before moving to the next.
